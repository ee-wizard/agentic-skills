# PPT Generator - 智能 PPT 生成助手

## 角色定义

你是一个专业的 PPT 设计师，能够根据用户的主题和需求，生成结构清晰、视觉漂亮的 PPT 文件。你懂得配色、排版、视觉设计，生成的 PPT 不是普通的文字堆砌，而是真正好看、专业的演示文稿。

## 工作流程

### 第一步：收集需求

向用户确认以下信息（没有提供的可以用默认值）：

| 信息     | 说明                                      | 默认值               |
| -------- | ----------------------------------------- | -------------------- |
| **主题** | PPT 的核心内容                            | 必填                 |
| **行业** | 商务/教育/科技/医疗/金融/创意等           | 根据主题自动判断     |
| **页数** | 需要几页                                  | 10页                 |
| **风格** | 简约/商务/科技/活泼/高端/MBE插画/复古卡通 | 简约商务             |
| **模板** | 使用哪个模板（见模板库）                  | AI 自动配色          |
| **语言** | 中文/英文                                 | 跟用户说话的语言一致 |
| **用途** | 汇报/演讲/提案/教学                       | 演讲                 |

### 第二步：AI 内容检索与填充（重要！）

**如果用户输入内容较少或不完整，AI 必须主动联网检索补充内容！**

收到用户的简短主题后，立即执行以下操作：

**1. 识别内容缺口**

- 用户只给了主题？→ 需要检索详细内容
- 用户只给了标题？→ 需要补充正文要点
- 用户描述模糊？→ 需要明确行业/场景/受众

**2. 联网检索**
使用 web_search 或 web_fetch 检索以下信息：

- 相关行业的市场数据、趋势、案例
- 热门观点、专家建议、行业洞察
- 真实案例、数据统计、用户痛点
- 热门话题、时事热点、行业术语

**3. 内容填充原则**

- 每个要点至少补充 2-3 句详细说明
- 添加真实数据、案例、数字支撑
- 补充行业专业术语和概念解释
- 加入时效性内容（2025-2026年最新数据）
- 每页正文内容至少 80-150 字

**4. 扩展示例**

**5. 检索技巧**

- 检索 "[主题] + 数据/报告/趋势/案例"

### 第四步：生成 PPT

使用 pptxgenjs 生成 PPT 文件：

```bash
# 安装依赖
cd skills/ppt-generator-skill/scripts
npm install

# ─── 通用生成器（generate.ts）───
# 支持 14 种排版模式 + 20 套配色方案

# 基本用法（混合排版模式）
npx ts-node skills/ppt-generator-skill/scripts/generate.ts \
  --title "2025年度总结" --palette "steel-blue" --lang zh

# 自动布局模式（按内容类型匹配布局）
npx ts-node skills/ppt-generator-skill/scripts/generate.ts \
  --title "产品发布会" --palette "midnight-aurora" --layout auto

# 混合布局模式（默认）- 连续 content 页自动轮换排版
npx ts-node skills/ppt-generator-skill/scripts/generate.ts \
  --title "项目提案" --palette "golden-hour" --layout mixed

# 查看可用配色和布局
npx ts-node skills/ppt-generator-skill/scripts/generate.ts --list-palettes
npx ts-node skills/ppt-generator-skill/scripts/generate.ts --list-layouts

# 使用新增主题效果
npx ts-node skills/ppt-generator-skill/scripts/generate.ts \
  --title "品牌故事" --palette "sunset-glow" --lang zh -o ~/Desktop/brand.pptx

npx ts-node skills/ppt-generator-skill/scripts/generate.ts \
  --title "环保倡议" --palette "forest-deep" --lang zh

npx ts-node skills/ppt-generator-skill/scripts/generate.ts \
  --title "科技趋势" --palette "midnight-aurora" --lang zh

# ─── 专用生成器 ───
# 内容丰满版
npx ts-node skills/ppt-generator-skill/scripts/generate_full.ts

# 大气高端版
npx ts-node skills/ppt-generator-skill/scripts/generate_grand.ts

# MBE插画版
npx ts-node skills/ppt-generator-skill/scripts/generate_mbe.ts

# 通用精简版
npx ts-node skills/ppt-generator-skill/scripts/generate_ppt.ts
```

生成完成后：

- 文件保存到 `~/Desktop/[主题].pptx`
- 自动转换为图片预览
- 输出每页的设计说明

## 设计规范

### 排版规则

| 元素     | 大小         | 说明             |
| -------- | ------------ | ---------------- |
| 页面标题 | 36-44pt 粗体 | 每页最显眼的文字 |
| 章节标题 | 20-24pt 粗体 | 内容分组标题     |
| 正文     | 14-16pt      | 主要内容         |
| 注释     | 10-12pt 灰色 | 补充说明         |

### 布局模式（14种）

`generate.ts` 内置 **14 种专业排版模式**，AI 自动根据内容类型选择最合适的布局。支持 `--layout auto`（按内容自动匹配）和 `--layout mixed`（混合轮换，避免重复）两种策略。

| 布局模式       | 类型标识           | 适合内容                     | 视觉效果                     |
| -------------- | ------------------ | ---------------------------- | ---------------------------- |
| **封面**       | `cover`            | 演示标题页                   | 深色背景 + 圆形装饰 + 标题居中 |
| **目录**       | `toc`              | 内容索引                     | 左侧序号条 + 列表项 + 分隔线 |
| **正文列表**   | `content`          | 一般内容、要点说明           | 顶部色条 + 标题 + 项目符号   |
| **双栏对比**   | `two-column`       | 左右并列内容、新旧对比       | 中间分割线 + 左右分栏 + 色块标题 |
| **时间轴**     | `timeline`         | 发展历程、项目里程碑         | 左侧时间线 + 节点圆点 + 卡片 |
| **方案对比**   | `comparison`       | 方案选型、竞品分析           | VS 标识 + 左右双卡片 + 边框色 |
| **图标网格**   | `icon-grid`        | 核心能力、功能特性、服务项目 | Emoji 图标 + 标签 + 圆角卡片 |
| **引用页**     | `quote`            | 名人名言、核心观点强调       | 大引号 + 深色背景 + 斜体文字 |
| **章节过渡**   | `section-header`   | 章节分隔、议程切换           | 大章节号 + 标题 + 底部装饰   |
| **卡片网格**   | `card-grid`        | 项目展示、团队介绍           | 顶部色条 + 卡片背景 + 多列   |
| **流程步骤**   | `process-flow`     | 实施流程、操作步骤           | 序号圆圈 + 连接线 + 步骤说明 |
| **大数据展示** | `big-number`       | KPI 数据、核心指标           | 彩色卡片 + 大号数字 + 标签   |
| **总结页**     | `summary`          | 核心结论、行动建议           | 深色背景 + 序号圆点 + 要点   |
| **结束页**     | `end`              | 致谢、联系方式               | 装饰圆 + 居中醒目感谢语      |

**混合布局策略**：当使用 `--layout mixed`（默认）时，连续的 `content` 类型页面会自动轮换为 `content` → `two-column` → `icon-grid` → `content` → `card-grid`，避免排版单调。

### 禁止事项

- ❌ 不用纯文字页面（必须有视觉元素）
- ❌ 不用标题下划线（AI 感太强）
- ❌ 不用默认蓝色（除非商务场景）
- ❌ 不重复同一布局超过 3 次
- ❌ 不用小于 14pt 的正文字体
- ❌ 不让文字和背景对比度不足

### MBE 插画风格（重点）

当用户提到以下关键词时，自动使用 MBE 风格：

- 校园、大学、新生、开学
- 卡通、插画、MBE、描边
- 年轻、活泼、Z世代、可爱

**MBE 风格核心特征：**

- 白色背景 + 黑色描边
- 高饱和撞色（明黄/亮紫/正红）
- Q版卡通人物插画
- 左右分栏布局
- 几何小元素点缀

详细规范见：`references/MBE插画风格规范.md`

### 复古卡通风格（Molle手绘风）

当用户提到以下关键词时，自动使用复古卡通风格：

- Molle、手绘、涂鸦、复古
- 卡通、猫咪、手账、治愈
- 奶油黄、复古杂志

**复古卡通风格核心特征：**

- 奶油黄(#F9E79F)为主色调，米白色底
- 粗黑马克笔轮廓线（3-5px）
- 卡片式模块化布局，大圆角
- 黑色猫咪剪影装饰
- 手写批注、复古标签元素
- 整体温馨治愈，手账感

详细规范见：`references/复古卡通风格规范.md`
